# Сайт риэлторского агентства

Сайт для риэлторов, позволяет вести учет квартир с подробной информацией, владельцев квартир, жалоб на квартиры в продаже. Интерфейс сайта позволяет производить быстрый поиск квартир по ключевым параметрам: город, адрес, владелец. В жалобах по автору и квартире, а владельца можно найти по номеру телефона или ФИО.

## Запуск

Для запуска сайта локально у Вас должен быть установлен Python3.

- Скачиваете репозиторий.
- Устанавливаете зависимости командой `pip install -r requirements.txt`
- Создаете файл базы данных и сразу применяете все миграции командой `python3 manage.py migrate`
- Запускаете сервер командой `python3 manage.py runserver`
- Зайти на локально запущенный сайт можно перейдя по адресу:
```
http://127.0.0.1:8000/
```
Для того чтобы попасть в панель администратора, необходимо завести учетную запись админа командой:
```
python manage.py createsuperuser
```
После исполнения командная строка попросит Вас ввести имя пользователя, почту и пароль. После этого зайти в административную панель локально запущенного сайта можно перейдя по адресу:
```
http://127.0.0.1:8000/admin/
```

## Переменные окружения

Часть настроек проекта берётся из переменных окружения. Чтобы их определить, создайте файл `.env` рядом с `manage.py` и запишите туда данные в таком формате: `ПЕРЕМЕННАЯ=значение`.

Доступны 3 переменные:
- `DEBUG` — дебаг-режим. Поставьте True, чтобы увидеть отладочную информацию в случае ошибки.
- `SECRET_KEY` — секретный ключ проекта
- `ALLOWED_HOSTS` — см [документацию Django](https://docs.djangoproject.com/en/5.2/ref/settings/#allowed-hosts).
- `DATABASE` — однострочный адрес к базе данных, например: `sqlite:///db.sqlite3`. Больше информации в [документации](https://github.com/jacobian/dj-database-url)

    Это позволяет легко переключаться между базами данных: PostgreSQL, MySQL, SQLite — без разницы, нужно лишь подставить нужный адрес.
